acerca de react

Es una librería de javascript para construir interfaces de usuario, es decir que es una librería solo para la vistas. Quieres usar ember?, angula?, backbone? o react.js? no hay problema, cuando cambian los datos cambias las vistas pero no a la inversa, los datos fluyen en un solo sentido, los componentes son el corazón de react.js.
Interfaces reutilizables y compuestas entre sí. El código HTML y el comportamiento ya no son cosas separadas nos deja escribir nuestros componentes con JSX. El problema principal es que el estado de nuestra app cambia todo el tiempo actualizaciones del server, interacciones del usuaria, eventos temporales, etc la solución es react.js y no se está hablando de un estado global de toda la aplicación cada componente tiene su propio estado cuando los datos cambian react.js vuelve a rendir izar las vistas en base a estos nuevos datos

¿como lo hace ?

Virtual DOM: es un árbol de objetos guardados en memoria, contiene una copia de lo que estamos viendo actualmente en el DOM y represente lo que estamos viendo en el DOM. Cuando un componente cambia sus datos react.js vuelve a dibujar la vista de este componente luego compara este resultado con lo que tiene almacenado en el virtual DOM, si son distintos recién allí es que toca al DOM pero no vuelve a dibujar toda vista del componente.

¿que es React.JS?

React.js es una libreria javascript desarrollada por facebook basada en componentes, ya que tus proyectos siempre tienen navegación, botones interfaces que se quieren reusar, react.js lo hace en serio. Cuando cambias los datos react.js actualiza todo, sin escribir más códigos sin recargar más componentes “magia”. Tus páginas se posicionan mucho mejor, en vez de tener “una plantilla vacía” react.js usa JSX que es una combinación entre HTML y javascript sin template ayudante a programar solo la parte divertida.

React.js is a javascript library developed by facebook and designed to help create SPA (single-page application), its objective is to try to fascinate the task of developing user interfaces. it should also be borne in mind that it makes use of devices called component-oriented programming.
